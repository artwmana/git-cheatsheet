```git ls-files | wc -l``` - проверяем кол-во активных файлов (желательно, чтобы было до 20k-30k)
```find . -type f | wc -l``` - кол-во всего файлов (желательно, чтобы было до 100k)
```du -sh .``` - размер репазитория

```git config core.untrackedCache true```- Git кеширует список файлов, чтобы не сканировать всё дерево каждый раз.
```git config core.fsmonitor true``` - Git начинает получать список изменённых файлов от файловой системы, а не сканирует весь репозиторий.
